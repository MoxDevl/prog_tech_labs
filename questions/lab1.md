# **Lab1**

1. ***Что такое Git и для чего он используется?***

    **Git** это один из VCS (Version Control System).
    Гит хранит снимки состояний проекта, где файлы хранятся в виде
    копий 

2. ***Как инициализировать новый репозиторий Git в существующей папке?***

    С помощью команды `git init`

3. ***Как добавить файлы в индекс (staging area) и сохранить изменения в локальный репозиторий?***

    Файлы добавляются с помощью `git add`, а `git commit` записывает измениния в репозиторий

4. ***Какая команда используется для просмотра текущего состояния рабочей директории?***

    `git status` показывает отличия между `HEAD` и индексом


5. ***Что такое ветка в Git, и как создать новую ветку?***

    Ветка это легковесный указатель, который можно двигать
    Новую ветку можно создать с помощью `git branch <name> [start_point]` или
    с помощью `git checkout -b <name> [start_point]`.
    Эти команды делают похожие вещи, но чуть-чуть отличаются


6. ***Как слить изменения из одной ветки в другую?***

    С помощью `git merge <commit>...` можно слить изменения из ветки
    `<commit>` в текущиую ветку. По результату слияния, будет создани отдельный коммит со слитыми файлами


7. ***Какая команда позволяет просматривать историю коммитов?***

    Удобная функция для просмотра `git log`
    Крутый опции:
    - `--stat`
    - `--graph`
    - `--pretty=oneline`

8. ***Что такое коммит в Git?***

    Коммит это сочетание снимков файлов

9. ***Что делает команда git add?***

    Добавляет измененные файлы в индекс из рабочей директории

10. ***Как узнать, на какой ветке ты находишься?***

    Выводит текущую ветку `git branch --show-current`

    Если в detached head, то ничего

11. ***Как переключиться на другую ветку?***

    `git checkout <branch_name>`

12. ***Как отправить ветку в удаленный репозиторий?***

    `git remote -u <origin> <branch>`

    `-u` обозначает `--set-upstream`, таким образом можно 
    обозначить tracking ветку

13. ***Что такое remote в Git?***

    С помощью `git remote` можно установить какие удаленные
    репозитории твой репозиторий отслеживает

    Например, вместо того, чтобы писать URL до удал. реп. каждый раз —
    можно поставить им имя
