14. ### *На какие виды делятся тесты по степени их автоматизации? К какой группе относятся Unit (модульные) и интеграционные тесты? Что они из себя представляют и кто их пишет (разработчик или тестировщик)?*

    Виды тестирования:
    - Автоматическое — _тестит софт_
    - Ручное — _тестишь руками_
    - Автоматизированное — _тестишь руками и тестит софт_

    Unit и модульное тестирования могут относится к любой из групп

15. ### *Раскройте понятие Unit (модульный) тест. Что значит “модульный”? Как решается проблема изоляции тестируемого кода от зависимостей в рамках модульного тестирования?*

    Во время модульного тестирования проверяются отдельные
    маленькие кусочки программы, то есть какие-то
    её маленькие составляющие.

    Один из способо избавления от зависимостей — создание mock-классов
    на место зависимостей, которые будут работать вместо обычного класса



## Если сдал >5 лаб

18. ### *Чем git pull отличается от git fetch? Для чего используется git clone?*

    `git pull` подгружает все коммиты из \<origin\> и делает мердж \<origin\>/\<branch\> в локальную \<branch\>

    `git fetch` лишь подргужает коммиты из \<origin\>

19. ### *Чем композиция (вид связи объектов в ООП) отличается от наследования? В каких паттернах проектирования используется композиция?*

    В композиции берется объект класса, чьи функции используются, и делают его
    элементов класса. Через этот элемент проходит в последствии вся логика.

    **Композиция используется в**:
    
    - Прокси
    - Декоратор
    - Цепь ответственности
    - Фасад

20. ### *Что такое ad-hoc полиморфизм? Чем он отличается от обычного полиморфизма (полиморфизм подтипов)?*

    **subtype polymorphism** это возможность использовать, например в c++, наследованные классы
    через указатель к базовому классу

    **parametric polymorphism** позволяет функциям запускать один и тот же код
    для любого типа аргументов

    **ad-hoc polymorphism** позволяет функциям действовать по-разному,
    в зависимости от поступающих аргументов
